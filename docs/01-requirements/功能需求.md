## 一、核心功能需求列表

### 1. 笔记管理模块
1. 可以上传学习笔记，格式包括图片、文档、手写的笔记和电子档都可以
2. 支持对笔记内容进行分类划分，每个分类之间支持设定关系，关系相关则代表分类之间有相关性，不相关则表示分类之间的知识是独立的
3. 支持对所有分类知识进行归纳总结，并输出分析报告

### 2. 脑图生成模块
4. 可以根据上传的笔记自动生成脑图
5. 支持对书本直接归纳总结，输出书本的脑图结构
6. 支持对照自己的脑图和书本直接输出的脑图的比较，帮助用于确定自己归纳不到位的地方

### 3. 知识点互动模块
7. 可以根据脑图进行知识点的提问
8. 针对脑图回答错误的地方，会自动记录，然后去笔记中找到相应的内容进行展示
9. 支持对笔记的追加提问和补充

### 4. 错题管理模块
10. 可以上传错题库，格式包括文档、照片和手写的笔记
11. 可以使用错题库进行反复学习，结合错题库和脑图，指出用户薄弱的知识点，提示用户加强这些地方的学习

### 5. 协作学习模块
12. 支持分享自己的笔记
13. 支持分享自己的错题库
14. 支持下载别人的笔记
15. 支持将别人的笔记加入到自己的笔记中来
16. 支持将别人的错题加入到自己的错题库中来

## 二、功能模块化说明

### 核心学习闭环
```
笔记上传 → AI脑图生成 → 知识点提问 → 错题记录 → 薄弱点分析 → 复习巩固
```

### 模块依赖关系
- **笔记管理** 是基础，为脑图生成提供数据源
- **脑图生成** 是核心，连接笔记、知识点提问和错题分析
- **知识点互动** 是差异化功能，实现主动式学习
- **错题管理** 与脑图联动，形成知识薄弱点精准定位
- **协作学习** 扩展学习资源，实现知识共享

## 三、开发优先级建议

### Phase 1 - MVP核心功能（0-3个月）
**目标**: 验证核心价值，建立早期用户群

1. ✅ 笔记上传与存储（支持图片、PDF、手写识别）
2. ✅ AI脑图自动生成（基于笔记内容）
3. ✅ 基于脑图的知识点提问
4. ✅ 错题库管理（多格式错题存储）
5. ✅ 错题→薄弱知识点自动分析
6. ✅ 基础分享与下载功能

**技术栈建议**:
- 前端: React/Next.js + Tailwind CSS
- 后端: Node.js/Express 或 Python/FastAPI
- AI模型: DeepSeek/通义千问（开源模型降低成本）
- 数据库: PostgreSQL + 向量数据库（ChromaDB/Pinecone）
- OCR: 百度OCR API

### Phase 2 - 差异化功能（3-6个月）
**目标**: 建立竞争壁垒，提升用户粘性

7. ✅ 教材脑图自动生成
8. ✅ 用户脑图 vs 教材脑图对比分析
9. ✅ 错题自动定位到笔记原文
10. ✅ 笔记智能合并（去重+分类适配）
11. ✅ 错题智能合并
12. ✅ 分类关系标签系统（相关/不相关）

### Phase 3 - 增强功能（6-12个月）
**目标**: 完善生态，提升变现能力

13. ✅ 智能复习提醒（艾宾浩斯遗忘曲线）
14. ✅ 学习数据分析与可视化
15. ✅ 考试模拟功能（基于错题库生成试卷）
16. ✅ 学习小组与社区功能

## 四、后续优化方向

### 4.1 学习进度与数据可视化
**补充价值**: 让学习效果可视化，增强用户成就感

```
- 学习时长自动记录（日/周/月统计）
- 知识点掌握度量化评分（0-100%）
- 学习曲线图表展示（使用 ECharts/D3.js）
- 学习目标达成率统计
- 知识点覆盖率热力图
```

**技术实现**:
- 前端使用 ECharts 或 Chart.js
- 后端记录用户行为数据（学习时长、答题正确率等）
- 定期生成学习报告（周报/月报）

---

### 4.2 智能复习系统
**解决痛点**: 学完就忘，复习效率低

```
- 基于艾宾浩斯遗忘曲线的复习提醒
  * 20分钟后、1小时后、9小时后、1天后、2天后、6天后、31天后
- 考前冲刺计划自动生成
- 重点知识点优先复习推荐
- Flashcard翻转卡片复习模式
```

**算法逻辑**:
```
记忆强度 = f(复习次数, 答题正确率, 时间间隔)
复习优先级 = (1 - 记忆强度) × 知识点权重
```

---

### 4.3 考试模拟功能
**用户刚需**: 考公/考研用户需要模拟真实考试场景

```
- 基于错题库生成个性化模拟试卷
  * 按知识点权重随机抽题
  * 难度自适应调整
- 限时考试模式（倒计时）
- 成绩预测与排名定位
- 考后详细分析报告
  * 薄弱知识点清单
  * 与历史成绩对比
  * 学习建议生成
```

**商业化价值**: 高频使用场景，可作为付费功能

### 4.4 学习效率分析
**目标**: 帮助用户优化学习方法

```
- 学习时段效率分析
  * 早晨 vs 下午 vs 晚上的学习效率对比
  * 最佳学习时段推荐
- 错题收敛速度分析
  * 同一知识点的重复错误次数
  * 知识点掌握速度评估
- 学习习惯分析
  * 连续学习天数打卡
  * 学习专注度评估（基于操作频率）
```

## 五、技术架构建议

### 前端架构
```
├── Web端 (React/Next.js)
│   ├── 笔记编辑器 (Tiptip/ProseMirror)
│   ├── 脑图可视化 (D3.js/AntV)
│   └── 数据可视化 (ECharts)
├── 移动端 (React Native/Flutter)
│   ├── 拍照上传
│   ├── 离线缓存
│   └── 推送提醒
└── 管理后台 (Ant Design Pro)
```

### 后端架构
```
├── API服务 (Node.js/Express 或 Python/FastAPI)
├── AI服务层
│   ├── 大模型调用 (DeepSeek/通义千问)
│   ├── 向量检索 (ChromaDB/Pinecone)
│   └── OCR服务 (百度/腾讯OCR API)
├── 数据存储
│   ├── 关系数据库 (PostgreSQL)
│   ├── 向量数据库 (ChromaDB)
│   ├── 对象存储 (阿里云OSS/AWS S3)
│   └── 缓存 (Redis)
└── 消息队列 (RabbitMQ/Kafka)
    * 异步处理AI任务
    * 定时复习提醒
```

### 核心技术难点
1. **知识图谱构建**: 如何从非结构化笔记中提取知识点关系
2. **脑图对比算法**: 如何计算两个脑图的相似度和差异
3. **实时协作**: 多人同时编辑笔记的冲突解决
4. **OCR准确率**: 手写笔记和公式的识别准确率
5. **AI成本控制**: 如何在保证效果的前提下降低API调用成本

## 七、风险与应对

### 7.1 技术风险
| 风险 | 应对策略 |
|------|----------|
| AI模型调用成本高 | 混合架构：基础功能用开源模型，高级功能用API |
| OCR识别准确率低 | 采用百度/腾讯OCR API，准确率可达95%+ |
| 知识图谱构建困难 | 结合规则引擎+大模型，逐步优化 |

### 7.2 市场风险
| 风险 | 应对策略 |
|------|----------|
| 竞品快速迭代 | 加速开发，突出差异化功能 |
| 用户付费意愿低 | 阶梯定价，提供试用期和学习成果可视化 |
| 获客成本高 | 社区化运营，口碑传播，降低CAC |

## 八、关键指标（KPI）

### 产品阶段
- **MVP阶段**: 月活用户(MAU) > 1000，留存率 > 30%
- **增长阶段**: MAU > 10000，付费转化率 > 5%
- **成熟阶段**: MAU > 50000，ARR（年度经常性收入）> 500万

### 核心指标
- **用户留存**: 次日留存 > 40%，7日留存 > 20%
- **使用深度**: 日均使用时长 > 20分钟
- **付费转化**: 免费到付费转化率 > 8%
- **用户满意度**: NPS > 40
